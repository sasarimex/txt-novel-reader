# 用户指南

## 安装

从 GitHub Releases 下载最新的 Windows `Setup.exe` 安装包，双击安装后启动软件。

## 首次使用

1. 打开软件。
2. 点击导入 TXT。
3. 选择一本或多本 TXT 小说。
4. 在书架中打开书籍开始阅读。

## 导入 TXT

支持单本导入和批量导入。软件会根据 TXT 内容生成 `bookHash`，用于本地识别和云端同步匹配。同一本 TXT 在不同设备导入后可以匹配到同一条云端进度。

## 书架和分类

- 可以编辑书名、作者和备注。
- 可以创建、重命名和删除分类。
- 可以把书籍拖拽到分类中。
- 删除分类不会删除书籍，书籍会回到未分类。

## 阅读设置

阅读界面支持调整字号、行高、主题和阅读方式。设置会保存在本地；登录同步服务器后，部分阅读设置也会随进度同步。

## 目录与书签

章节目录由本地解析器从 TXT 内容中生成。阅读时可以添加书签，书签会记录位置、章节和预览文本。

## 云端同步

云端同步需要先部署服务器端项目，然后在客户端填写服务器根地址。例如：

```text
https://read.example.com
```

请不要填写 `/api`。客户端会自动拼接 API 路径。

同步内容包括：书架元数据、阅读进度、书签、阅读设置和分类。服务器不保存 TXT 正文。

## 更换同步服务器

更换服务器会清空当前同步登录状态，需要使用新服务器上的账号重新登录。更换前请确认本地书库仍在。

## 常见问题

### 为什么不上传 node_modules？

依赖可以通过 `npm install` 重新安装，上传 `node_modules` 会让仓库非常大。

### 为什么云端书籍打开后提示需要导入同一本 TXT？

服务器不保存 TXT 正文。新设备需要手动导入同一本 TXT，客户端会使用内容哈希匹配云端进度。

### 同步连接失败怎么办？

确认服务器地址是根地址，确认服务器 `/api/health` 可访问，检查 CORS、HTTPS 证书和服务器日志。
