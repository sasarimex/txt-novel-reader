# 摸鱼神器 开发任务拆分

版本：P0 v0.8  
日期：2026-05-28  
平台：Windows only  

## 1. 开发里程碑

建议按 7 个里程碑推进：

- M0：项目初始化
- M1：基础桌面壳和页面骨架
- M2：书架、导入、删除
- M3：编码识别和章节解析
- M4：阅读器基础体验
- M5：互斥翻页模式和页码
- M6：持久化、排序、边界状态
- M7：测试、打包、验收

## 2. M0 项目初始化

目标：

- 创建 Electron + React + TypeScript + Vite 项目。
- 配置基础开发环境。

任务：

- 初始化 package.json。
- 配置 Vite。
- 配置 Electron main 和 preload。
- 配置 TypeScript。
- 配置 ESLint 或基础格式化工具。
- 配置开发启动命令。
- 配置 Windows 打包脚本。

交付物：

- 可以运行开发窗口。
- Renderer 页面可以显示 `摸鱼神器`。
- Main、Preload、Renderer 三层可以正常通信。

验收：

- `npm run dev` 可以启动桌面窗口。
- 页面加载无白屏。
- 控制台无关键报错。

## 3. M1 基础桌面壳和页面骨架

目标：

- 搭建书架页、阅读器页和基础路由。

任务：

- 创建 `BookshelfPage`。
- 创建 `ReaderPage`。
- 创建顶部栏组件。
- 创建基础按钮、弹窗、确认框组件。
- 创建全局样式和主题变量。
- 创建全局设置 store。

交付物：

- 书架页可见。
- 阅读器页可通过假数据进入。
- 目录弹窗和设置弹窗可以打开关闭。

验收：

- 点击书籍假卡片可以进入阅读器。
- 点击返回可以回到书架。
- 点击目录按钮弹出居中目录弹窗。
- 点击齿轮弹出设置弹窗。

## 4. M2 书架、导入、删除

目标：

- 实现真实书架数据。
- 支持导入 txt 并复制到软件管理目录。
- 支持删除书籍。

任务：

- 定义 `Book` 数据模型。
- 建立 app data 目录结构。
- 实现 `library.json` 读写。
- 实现 `settings.json` 读写。
- 实现 `books.list()` IPC。
- 实现 `books.importTxt()` IPC。
- 导入时复制 txt 到 `books/{bookId}/source.txt`。
- 导入成功后留在书架。
- 书架卡片显示书名和进度。
- 实现删除书籍二次确认。
- 删除软件管理目录中的书籍副本和元数据。

交付物：

- 用户可以导入 txt。
- 导入后书架出现书名卡片。
- 用户可以删除书籍。

验收：

- 导入后原始 txt 文件不被移动或删除。
- 删除书籍不会影响原始 txt。
- 空书架显示空状态。
- 删除前必须出现确认弹窗。

## 5. M3 编码识别和章节解析

目标：

- txt 导入后可正确读取中文内容。
- 自动识别 `第一卷` 和 `第一章`。

任务：

- 实现编码识别模块。
- 支持 UTF-8。
- 支持 UTF-8 BOM。
- 支持 GBK/GB18030。
- 导入后生成 UTF-8 `normalized.txt`。
- 实现章节解析模块。
- 实现分卷解析模块。
- 生成 `catalog.json`。
- 支持无目录状态。
- 为解析模块添加单元测试。

交付物：

- 阅读器能读取 normalized 文本。
- 目录弹窗能展示解析后的章节和分卷。

验收：

- UTF-8 小说导入后无乱码。
- GBK/GB18030 小说导入后无明显乱码。
- `第一卷` 能作为目录分组。
- `第一章` 能作为章节。
- 没有章节时仍能打开阅读。

## 6. M4 阅读器基础体验

目标：

- 阅读器可以稳定显示正文、目录、设置、主题和字号。

任务：

- 实现 `books.readText(bookId)`。
- 实现 `books.readCatalog(bookId)`。
- 阅读器加载正文。
- 正文区域居中显示。
- 页码区域布局到右下角。
- 实现目录弹窗点击章节跳转。
- 实现设置弹窗。
- 实现字号调整，范围 16px 到 32px，默认 20px。
- 实现三种主题。
- 护眼主题使用背景 `#DCE8D2` 和文字 `#243024`。

交付物：

- 用户可以打开一本书并阅读。
- 用户可以调字号和主题。
- 用户可以打开目录并跳转章节。

验收：

- 字号调整立即生效。

## 7. M5 互斥翻页模式和页码

目标：

- 实现滚轮模式和点击翻页模式。
- 两种模式互斥。
- 两种模式都显示页码。

任务：

- 定义 `pageMode` 设置。
- 设置弹窗中加入翻页模式分段控件。
- 实现滚轮模式。
- 滚轮模式下左右点击无效。
- 滚轮模式下根据滚动位置计算页码。
- 实现点击翻页模式。
- 点击翻页模式下启用左右半屏点击区域。
- 点击翻页模式下阻止滚轮默认行为。
- 点击翻页模式下正文不滚动。
- 点击翻页模式下计算真实分页页码。
- 弹窗和顶部栏点击不触发翻页。
- 模式切换时按阅读进度恢复位置。

交付物：

- 阅读器设置可切换两种模式。
- 当前只允许一种翻页方式生效。

验收：

- 滚轮模式下滚轮能移动正文。
- 滚轮模式下点击左半屏或右半屏不会翻页。
- 点击翻页模式下点击左半屏上一页。
- 点击翻页模式下点击右半屏下一页。
- 点击翻页模式下滚轮不会移动正文。
- 页码始终显示在右下角。

## 10. 推荐开发顺序

1. 先跑通 Electron 框架。
2. 先做书架假数据和阅读器假数据。
3. 再接入真实导入和本地存储。
4. 再做编码识别和章节解析。
5. 再做阅读器分页。
6. 最后处理重复导入、删除、排序和打包。

理由：

- 阅读器分页是 P0 中技术风险最高的部分。
- 但分页依赖真实正文和设置系统。
- 先打通数据流，再精调阅读体验，整体风险更低。

## 12. 主要风险和应对

### 风险 1：分页不稳定

应对：

- 用 `progressRatio` 作为跨布局恢复依据。
- 页码变化时允许近似恢复，不追求绝对逐字一致。

### 风险 2：编码识别不准

应对：

- BOM 优先。
- UTF-8 严格校验。
- GB18030 兜底。
- 乱码比例检查。

### 风险 3：大文件渲染卡顿

应对：

- 导入时缓存解析结果。
- 阅读器避免频繁重排。
- 阅读位置保存节流。
- 如 P0 后期性能不足，再考虑虚拟化渲染。

### 风险 4：误删或误覆盖

应对：

- 删除必须二次确认。
- 重复导入必须明确跳过或覆盖。
- 删除只删除软件管理目录副本，不触碰原始文件。
